# /tdd — 仕様駆動 TDD オーケストレータ（{{PROJECT_NAME}}）

> あなたは振る舞いやテストを考えなくてよい。**仕様書を渡せば、テストを抽出して先に書き、それに準じて実装し、検証まで**回す。
> 言語: TypeScript ／ unit: Vitest ／ E2E: Playwright ／ アプリ: {{DOMAIN_SUMMARY}}

## 入力
`$ARGUMENTS` = 仕様書のパス（`.claude/tdd/specs/<slug>.md`）またはテキスト。
仕様が無い・固まっていないなら、**まず `/spec` を促す**（テストにできない曖昧仕様で着手しない）。

## 着手前に必ず読む
- `.claude/tdd/rules/tdd-flow.md` … 仕様駆動TDDの規律
- `.claude/tdd/rules/test-conventions.md` … unit/E2E の配置・命名・モック方針
- `.claude/tdd/test-strategy.md` … どの要件をどのレベル（unit/integration/E2E）で testするか
- `.claude/tdd/commands.md` … 実コマンド
- 渡された仕様書、および実装対象に最も近い既存コード/既存テスト

## フロー（この順に駆動する）

### 1. 仕様からテストを洗い出す
仕様の**受け入れ条件**を、検証可能なテストケースに分解する。各ケースに**レベルを付与**（`test-strategy.md` 準拠）:
- **E2E（Playwright）** … ユーザー操作フロー・画面をまたぐ受け入れ条件
- **integration** … 複数モジュール/route handler/DB 等をまたぐ
- **unit（Vitest）** … 純粋ロジック・関数・コンポーネント単体
正常 / 境界 / 異常を各条件から起こす。**E2E シナリオの洗い出しを最初に行う**（受け入れ全体の地図になる）。

### 2. 実装計画を立てる
触るファイル・追加/変更する関数・データの流れ・依存を、{{SRC_LAYOUT}} を踏まえて列挙する。
テストケース ↔ 実装箇所の対応表を作る。**この計画とテスト一覧を箇条書きで提示**（1回だけの目視確認。問題なければ即進行）。

### 3. テストを先に書く → テストをレビューする
- 計画したテストを**実装より先に**書く（unit/E2E とも `test-conventions.md` の配置・命名に従う）。
- 書いたら実行し、**期待どおり赤**になることを確認（未実装ゆえの失敗か、テストのミスかを切り分け）。
- **テスト自体をレビュー**: 受け入れ条件の取りこぼし・過剰・重複が無いか。仕様の言葉でテスト名になっているか。

### 4. 仕様内容を実装する
赤を緑にする実装を書く。最小実装→整える（リファクタ）。テストの意図に**準じて**実装し、テストを実装に合わせて歪めない。

### 5. 既存テストが通るか（回帰）
unit の全実行で、既存テストを壊していないか確認する（`commands.md` の全テストコマンド）。

### 6. E2E / 各種テストが通過するか
Playwright を含め、対象レベルのテストを通す。E2E 基盤が未導入で必要になったら、**導入可否を1度確認**してから入れる。

### 7. 一通りの動作確認
受け入れ条件を満たすか通しで確認する。必要なら dev 起動や主要フローの手動/自動確認（E2E でカバーしきれない部分）。

### 8. テスト実行時の不具合をレビュー
落ちたテスト・実行時の失敗があれば、**原因をレビュー**（実装バグか / テストが不正確か / 仕様の見落としか）。
修正 → 再実行 → 緑を確認。仕様レベルの抜けが見つかったら仕様書にも反映する。

### 仕上げ
- `.claude/tdd/progress.md` に、対応した仕様・テスト・結果を1エントリ追記。
- 受け入れ条件の単位でコミット（コミット規約はプロジェクトに従う）。`push` はユーザーが明示したときだけ。

---

## ルール
- **テストを先に書く。** 実装は失敗するテストを通すためだけに書く。テストを実装に合わせて歪めない。
- **緑でない状態でコミットしない。**
- コマンドは `.claude/tdd/commands.md` のものだけを使う（推測で別コマンドを叩かない）。
- 1 コミット = 1 受け入れ条件（または論理単位）。大きいと感じたら割る。
- 確認ステップ（2・6）は最小限。それ以外はユーザー確認を待たず自己完結で進める。
